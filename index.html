パート1：Pythonのきほん（レッスン1〜5）

やり方：各レッスンの「▶ このコードを試す」を押す → 下のコード欄に入るので「実行」→ 結果を見る → 「変えてみよう」に挑戦。この繰り返しで覚えます。

レッスン1：print（表示する）

print("こんにちは")
print(500)
解説：print( ) はカッコの中身を画面に表示する命令。文字は " " で囲む。数字はそのまま書く。
変えてみよう：自分の班の番号と「〇班です」を表示してみよう。

レッスン2：変数（値に名前をつける）

madogawa = 750     # 窓側の明るさ
rokagawa = 300     # 廊下側の明るさ
print(madogawa - rokagawa)   # 差
print(madogawa / rokagawa)   # 何倍か
解説：名前 = 値 で変数を作る。# から後ろはメモ（コメント）で、実行されない。- は引き算、/ は割り算。
変えてみよう：値を自由に変えて「何倍か」がどう変わるか確かめよう。これがSTEP3でやる「仮説：窓側は廊下側の◆倍明るい」の計算そのもの。

レッスン3：リスト（値をまとめて入れる）

values = [512, 530, 498, 505, 521]
print("データの数:", len(values))
print("合計:", sum(values))
print("平均:", sum(values) / len(values))
print("最大:", max(values), " 最小:", min(values))
解説：[ ] がリスト。センサーの計測値のような「たくさんの数」をまとめて扱える。len＝個数、sum＝合計、max/min＝最大/最小。
変えてみよう：リストに値を2つ追加して、平均がどう変わるか見てみよう。

レッスン4：for（くり返す）

values = [512, 530, 498]
for v in values:
    print("計測値:", v)
解説：for v in values: は「リストから1つずつ取り出して v と呼び、下の字下げした行をくり返す」。180行の計測データも9班ぶんのファイルも、これで一気に処理できる。字下げ（スペース4つ）が必須。
変えてみよう：くり返しの中で v * 2 を表示してみよう。

レッスン5：if（条件で分ける）

akarusa = 450
if akarusa >= 500:
    print("明るい（基準クリア）")
else:
    print("暗い（基準以下）")
解説：if 条件: が成り立てば上、成り立たなければ else: の下が実行される。>= は「以上」。学校環境衛生の基準では教室・黒板の照度は 500ルクス以上が望ましいとされている——この判定を自動化するのがSTEP3のミッション。
変えてみよう：akarusa の値をいろいろ変えて、表示が切り替わるのを確認しよう。

レッスン6：グラフ化 —— 数字の列を「見える」ようにする

解説：180個の数字を眺めても傾向は分からない。plt.plot(values) で折れ線グラフにすると、一瞬で「増えている／減っている／波打っている」が見える。これが可視化（情報Ⅰ：データの活用）。
変えてみよう：FILE を "data/rei_kumori.csv"（曇りの日）に変えて、形の違いを見よう。
読み取ったこと（グラフから分かった傾向を1行で）

レッスン7：くらべる —— 2つのデータを重ねる

解説：グラフの真価は比較にある。晴れの日と曇りの日を同じグラフに重ねると、差の大きさが一目で分かる。label= と plt.legend() で凡例（どの線がどれか）を付けるのがマナー。
変えてみよう：「何倍明るいか」を print で表示する行を追加しよう（ヒント：レッスン2の割り算）。
読み取ったこと

レッスン8：ばらつき —— 平均が同じでも「別物」

解説：データAとBは平均がほぼ同じ500。でもグラフにすると全く違う。安定した照明（A）とチカチカ変動する照明（B）では快適さが違うように、平均だけではデータは語れない。ばらつきの一番簡単な指標が「範囲＝最大−最小」（情報Ⅰ：基礎的な指標。情報Ⅱでは分散・標準偏差に発展）。
変えてみよう：AとBそれぞれの「範囲（max - min）」を print しよう。
読み取ったこと

レッスン9：異常値 —— 汚れたデータをそうじする

解説：本物のセンサーデータには異常値が混ざる（センサーを手でふさぐと0、通信エラーで9999など）。異常値が混ざると平均が大きく狂う。if 0 < v < 2000: のような条件で「ありえない値」を除いてから計算するのが前処理。STEP2の実測でも必ず役に立つ（情報Ⅰ：データの妥当性・限界／情報Ⅱ：データ処理）。
変えてみよう：除去する条件を変える（例：0 < v < 1000）と結果がどうなる？きびしすぎる条件の危険も考えよう。
読み取ったこと

レッスン10：シミュレーション —— 計算で「予測」する

解説：「ブラインドを閉めると明るさは30%になる」というモデル（仮定）を立てると、実測しなくても予測グラフが計算で作れる。これがモデル化とシミュレーション（情報Ⅰの重要単元）。STEP2で本当に計測して、予測と現実のズレを確かめるのがこの授業のクライマックス。
変えてみよう：割合 0.3 を変えて予測を調整しよう。「照明を1本追加（+200）」のモデルも作れる？
読み取ったこと
