レッスン6：個数・合計・最大・最小・平均・標準偏差

import math

values = [512, 530, 498, 505, 521]
heikin = sum(values) / len(values)

gokei = 0
for v in values:
    gokei = gokei + (v - heikin) ** 2

bunsan = gokei / len(values)      # 分散
hyojun = math.sqrt(bunsan)        # 標準偏差
print("標準偏差:", round(hyojun, 2))
解説：リストから len（個数）・sum（合計）・max/min（最大/最小）を求める。平均は 合計 ÷ 個数。
標準偏差＝ばらつきは、①平均との差を2乗して、②その平均をとり（＝分散）、③ルートをとる。数学Iで習う定義そのままを for で書ける。平均が同じでも、ばらつきが違えばデータの性格はまったく違う。この「平均」と「ばらつき」の2つが分析の土台で、STEP2でこれを図にする。
※ここでは仕組みを理解するために手で計算する。STEP2からは同じことを pandas が1行でやってくれる。
変えてみよう：リストを [100, 900, 100, 900, 100] のようにバラバラにすると標準偏差はどうなる？ 逆に全部同じ数字にすると（答え：0）？

レッスン10：ヒストグラム —— ばらつき（標準偏差）を図にする

解説：ヒストグラム（plt.hist）は「その値が何回出たか」を数えて棒にした図。レッスン6で手計算した標準偏差＝ばらつきが、そのまま形になって見える。山がとがっている＝標準偏差が小さい＝安定、横に広がっている＝標準偏差が大きい＝バラバラ。bins（棒の数）を変えると見え方が変わる。
pandasでは df["value"].std(ddof=0) と書く。ddof=0 を忘れると ÷(n−1) になり、レッスン6の手計算と数字が合わないので注意。
変えてみよう：bins を 4 や 20 に変えると形はどう変わる？ 印象が変わりすぎないか確かめよう。
📊 着眼点：印刷された標準偏差の数字と、山の広がりを見くらべよう。「ばらつきが大きい」を、数字と図の両方で言えるようになるのがゴール。ddof=0 を外すと数字がどう変わるかも試してみよう。
