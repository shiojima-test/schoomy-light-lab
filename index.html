パート1：Pythonのきほん（レッスン1〜5）

やり方：各レッスンの「▶ このコードを試す」を押す → 下のコード欄に入るので「実行」→ 結果を見る → 「変えてみよう」に挑戦。この繰り返しで覚えます。

レッスン1：print（表示する）

print("こんにちは")
print(500)
解説：print( ) はカッコの中身を画面に表示する命令。文字は " " で囲む。数字はそのまま書く。
変えてみよう：自分の班の番号と「〇班です」を表示してみよう。

レッスン2：変数（値に名前をつける）

madogawa = 750     # 窓側の明るさ
rokagawa = 300     # 廊下側の明るさ
print(madogawa - rokagawa)   # 差
print(madogawa / rokagawa)   # 何倍か
解説：名前 = 値 で変数を作る。# から後ろはメモ（コメント）で、実行されない。- は引き算、/ は割り算。
変えてみよう：値を自由に変えて「何倍か」がどう変わるか確かめよう。これがSTEP3でやる「仮説：窓側は廊下側の◆倍明るい」の計算そのもの。

レッスン3：リスト（値をまとめて入れる）

values = [512, 530, 498, 505, 521]
print("データの数:", len(values))
print("合計:", sum(values))
print("平均:", sum(values) / len(values))
print("最大:", max(values), " 最小:", min(values))
解説：[ ] がリスト。センサーの計測値のような「たくさんの数」をまとめて扱える。len＝個数、sum＝合計、max/min＝最大/最小。
変えてみよう：リストに値を2つ追加して、平均がどう変わるか見てみよう。

レッスン4：for（くり返す）

values = [512, 530, 498]
for v in values:
    print("計測値:", v)
解説：for v in values: は「リストから1つずつ取り出して v と呼び、下の字下げした行をくり返す」。180行の計測データも9班ぶんのファイルも、これで一気に処理できる。字下げ（スペース4つ）が必須。
変えてみよう：くり返しの中で v * 2 を表示してみよう。

レッスン5：if（条件で分ける）

akarusa = 450
if akarusa >= 500:
    print("明るい（基準クリア）")
else:
    print("暗い（基準以下）")
解説：if 条件: が成り立てば上、成り立たなければ else: の下が実行される。>= は「以上」。学校環境衛生の基準では教室・黒板の照度は 500ルクス以上が望ましいとされている——この判定を自動化するのがSTEP3のミッション。
変えてみよう：akarusa の値をいろいろ変えて、表示が切り替わるのを確認しよう。

レッスン7：くらべる —— 晴れとくもりを重ねる

解説：グラフの真価は比較。晴れ（6/1）とくもり（6/4）を同じグラフに重ねると、天気による光の差が一目で分かる。label= と plt.legend() で凡例（どの線がどれか）を付けるのがマナー。
変えてみよう：「晴れはくもりの何倍か」を print で表示する行を追加しよう（ヒント：レッスン2の割り算。合計どうしで割るとよい）。
読み取ったこと

レッスン8：ばらつき —— 「安定して晴れる月」はどっち？

解説：甲府の1月と6月（梅雨）、毎日の日照時間をくらべる。平均だけでなくばらつきを見るのがポイント。ばらつきの一番簡単な指標が「範囲＝最大−最小」。範囲が小さい＝毎日似ている＝安定、範囲が大きい＝日によって全然違う。
変えてみよう：範囲（max - min）をそれぞれ print して、どちらの月が「安定して晴れる」か数値で言えるようにしよう。
読み取ったこと

レッスン9：前処理 —— 「夜の0」をどうする？

解説：晴れの日（6/1）の平均を24時間分で計算すると、夜の0がたくさん混ざって小さく出る。「昼の光の強さ」を知りたいなら、if文で v > 0 のときだけ集めて計算し直す。目的に合わせてデータを整えるのが前処理。本物のデータには欠測（×や--の記号）もあり、そのまま計算すると狂う——STEP2の実測でも必ず使う考え方。
変えてみよう：条件を v >= 1.0（強い光の時間だけ）にすると平均はどうなる？条件のかけ方で「答え」が変わる危険も考えよう。
読み取ったこと

レッスン10：シミュレーション —— 計算で「予測」する

解説：「くもりの日の光は晴れの日の30%くらいになる」というモデル（仮定）を立てると、実測しなくても予測グラフが計算で作れる。これがモデル化とシミュレーション。予測と本物のくもりデータ（6/4）を重ねて、モデルの当たり具合を確かめよう。
変えてみよう：割合 0.3 を調整して、本物のくもりに一番近づく割合を探そう。それがあなたの作った「甲府のくもりモデル」！
読み取ったこと

レッスン11：データ推理ゲーム —— この日の天気を当てろ

解説：6月のある3日間（日付ひみつ）の本物の日射データがある（nazoA〜C）。グラフの形からその日の甲府の天気の変化を推理しよう。「午前は？午後は？急な変化はいつ？」——データ分析の最終目標は数値を出すことではなく、現実に何が起きたかを読み取ること。
